# 喝水提醒小程序 - 打包说明

## 打包步骤

### 1. 安装 PyInstaller
```bash
pip install pyinstaller
```

### 2. 打包程序
在项目目录下执行以下命令：
```bash
pyinstaller --onefile --windowed --icon=favicon.ico --name=喝水提醒 drink.py
```

参数说明：
- `--onefile`: 打包成单个 exe 文件
- `--windowed`: 不显示控制台窗口
- `--icon`: 指定程序图标（可选）
- `--name`: 指定 exe 文件名

### 3. 整理文件
打包完成后，在 `dist` 目录下会生成 `喝水提醒.exe`。

**重要：** 需要将以下文件复制到 `dist` 目录下（与 exe 文件同一目录）：
- `hydration_config.ini` - 配置文件
- `normal.gif` - 正常状态 GIF 图
- `touchhead.gif` - 提醒时 GIF 图
- 其他可能用到的 GIF 文件

### 4. 目录结构
最终的 `dist` 目录应该包含：
```
dist/
├── 喝水提醒.exe
├── hydration_config.ini
├── normal.gif
├── touchhead.gif
└── (其他 GIF 文件)
```

## 路径处理说明

程序已经做了适配，无论是源码运行还是打包后的 exe，都能正确获取程序所在目录：

### 源码运行时
- 程序目录：`drink.py` 所在的目录
- 配置文件：`./hydration_config.ini`
- GIF 文件：`./normal.gif`、`./touchhead.gif`

### exe 运行时
- 程序目录：`喝水提醒.exe` 所在的目录
- 配置文件：`./hydration_config.ini`
- GIF 文件：`./normal.gif`、`./touchhead.gif`

## 配置文件说明

`hydration_config.ini` 中的 GIF 路径只需要填写文件名即可，例如：
```ini
gif_path = normal.gif
reminder_gif_path = touchhead.gif
```

程序会自动在 exe 所在目录下查找这些文件。

## 注意事项

1. **必须将依赖文件与 exe 放在同一目录**
   - 配置文件：`hydration_config.ini`
   - GIF 文件：`normal.gif`、`touchhead.gif` 等

2. **不要将依赖文件打包进 exe**
   - 这些文件需要保持独立，方便用户修改配置
   - 程序会在运行时自动读取这些文件

3. **首次运行**
   - 如果 exe 目录下没有配置文件，程序会自动创建默认配置
   - 建议手动提供配置文件，以便自定义设置

## 分发程序

将 `dist` 目录下的所有文件（exe、ini、gif 等）一起打包成 zip 文件，分发给用户即可。

用户只需解压后运行 `喝水提醒.exe` 即可。

## 开机自启动功能

程序支持开机自动启动功能，通过修改配置文件即可启用：

### 启用开机自启动

编辑 `hydration_config.ini` 文件，将 `auto_start` 设置为 `1`：

```ini
# 开机自启动 (0 为不启用，1 为启用)
auto_start = 1
```

### 禁用开机自启动

将 `auto_start` 设置为 `0` 即可禁用：

```ini
# 开机自启动 (0 为不启用，1 为启用)
auto_start = 0
```

### 工作原理

程序通过修改 Windows 注册表实现开机自启动：
- 注册表路径：`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`
- 启动项名称：`HydrationReminder`
- 启动项值：程序的完整路径

### 注意事项

1. **打包后的 exe 程序**
   - 启用开机自启动后，程序会将 exe 的路径写入注册表
   - 请勿随意移动 exe 文件的位置，否则开机自启动会失败
   - 如需移动位置，请先禁用开机自启动，移动后重新启用

2. **源码运行**
   - 源码运行时，会使用 `python 脚本路径` 的方式启动
   - 需要确保 Python 环境可用

3. **权限问题**
   - 程序使用当前用户权限修改注册表，不需要管理员权限
   - 如果设置失败，请检查是否有足够的权限

4. **查看开机启动项**
   - 可以通过任务管理器 -> 启动 查看开机启动项
   - 也可以运行 `shell:startup` 打开启动文件夹查看
